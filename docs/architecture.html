terraform-aws-alb Architecture

HCL / Terraform AWS Infrastructure / IaC

Executive Summary

This module provisions a production-ready AWS Application Load Balancer with integrated security groups, multiple target groups, HTTP/HTTPS listeners with automatic TLS redirect, listener rules for content-based routing, S3 access logging with lifecycle management, and optional WAF Web ACL association. It enables teams to deploy enterprise-grade load balancing infrastructure in minutes, ensuring high availability, TLS 1.3 termination, and centralized traffic management across multi-service architectures.

Data Flow

  1. Client sends HTTP/HTTPS request to the ALB DNS name or custom domain.
  2. The Security Group evaluates the request against ingress CIDR and security group rules.
  3. If WAF is associated, the request is inspected for malicious patterns before reaching the ALB.
  4. The HTTP listener on port 80 issues a 301 redirect to HTTPS (when a certificate is configured), or forwards directly to the default target group.
  5. The HTTPS listener terminates TLS using the ACM certificate (TLS 1.3 policy) and evaluates listener rules.
  6. Listener rules match on host headers, path patterns, or HTTP headers to route to the appropriate target group.
  7. The target group forwards the request to registered targets (EC2, ECS, IP) and performs periodic health checks.
  8. Session stickiness (lb_cookie) is optionally applied for stateful applications.
  9. Access logs are written to S3 with lifecycle management (30-day IA transition, 90-day expiration).

Security Controls

  • TLS 1.3 Termination — Default SSL policy ELBSecurityPolicy-TLS13-1-2-2021-06 enforces modern cipher suites.
  • HTTP-to-HTTPS Redirect — Automatic 301 redirect eliminates unencrypted traffic.
  • Security Groups — Configurable ingress CIDRs and security group source rules restrict access.
  • Invalid Header Dropping — drop_invalid_header_fields enabled by default to prevent header-based attacks.
  • WAF Integration — Optional AWS WAFv2 Web ACL association for OWASP Top 10 protection.
  • Deletion Protection — Optional flag prevents accidental ALB deletion.
  • S3 Access Logs — Encrypted log storage with lifecycle policies for compliance.
  • SNI Certificates — Support for multiple SSL certificates via Server Name Indication.

Production Readiness Checklist

  • Enable HTTPS with a valid ACM certificate (certificate_arn)
  • Set enable_deletion_protection = true to prevent accidental deletion
  • Enable access logging (enable_access_logs = true) for audit trails
  • Associate a WAF Web ACL (waf_arn) for DDoS and bot protection
  • Restrict ingress_cidr_blocks to known IP ranges (avoid 0.0.0.0/0 in production)
  • Configure health check paths per target group to match application endpoints
  • Set appropriate deregistration_delay for graceful connection draining
  • Enable drop_invalid_header_fields (enabled by default)
  • Use internal = true for private-facing services
  • Deploy across multiple subnets in different Availability Zones

Deployment

Quick Start

module "alb" {
  source = "github.com/kogunlowo123/terraform-aws-alb"

  project_name = "my-app"
  environment  = "prod"
  vpc_id       = "vpc-0123456789abcdef0"
  subnet_ids   = ["subnet-aaa", "subnet-bbb"]

  certificate_arn = "arn:aws:acm:us-east-1:123456789012:certificate/abc-123"

  target_groups = [
    {
      name = "web"
      port = 8080
    }
  ]
}
